# Audience and Decisions

Plan repositories around the reader's decisions, not only around topic clusters.

## Common audience types
- `Commercial / BD`: wants market-entry logic, buyer classes, priority labels, and stop conditions.
- `Leadership`: wants portfolio priorities, risk framing, and implications.
- `Operating / compliance`: wants certification, traceability, documentation, and access constraints.

## Decision-supported pattern
Every file should answer:
- What decision does this file support?
- What would the reader do differently after reading it?

## For commercial / BD readers
Ensure the repository can answer:
- Is this opportunity worth pursuing?
- Who buys it?
- Why now?
- What must be true first?
- What makes it a no-go?
- What is the better alternative?
