# Strategy Layer Patterns

Strategy writing is different from reference writing.

## Every major strategy file should answer
- What is the recommendation?
- Why now?
- Who buys or monetizes it?
- What must be true first?
- What would make us stop?
- What stronger alternative exists?

## Recommendation labels
Use a stable vocabulary:
- `Priority 1`
- `Priority 2`
- `Selective`
- `Monitor`
- `No-go`

Explain the label in prose before using it as shorthand.

## Anti-patterns
- Do not hide the recommendation until the end.
- Do not write generic “growth opportunity” language.
- Do not describe upside without a real stop condition.
- Do not omit evidence confidence when uncertainty matters.
