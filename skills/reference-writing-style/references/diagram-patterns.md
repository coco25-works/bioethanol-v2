# Diagram Patterns

Use diagrams before dense content.

## Common diagram types
- source stack
- market architecture
- pathway landscape
- analytical framework
- strategy decision logic
- repository navigation
- update workflow

## Diagram rules
- add a short prose caption before or after the diagram;
- keep labels short;
- prefer three to five boxes or stages over overly dense layouts;
- make the flow readable on a narrow terminal.

## When not to diagram
- when a single sentence is clearer;
- when the diagram would duplicate a tiny table exactly;
- when the relationship is already obvious from the section heading and opening paragraph.
