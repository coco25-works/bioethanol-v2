# Diagram Drawing

Use this skill when the document needs a visual frame before the dense prose or table.

## Workflow
1. Decide what the reader needs to understand first.
2. Choose the diagram type.
3. Decide whether the document needs one diagram or a connected series of diagrams.
4. Keep the flow to three to five main stages where possible.
5. Add a short caption or introductory sentence.
6. Place the diagram before the table or deep explanation it supports.

## Core Rules
- Optimize for readability in plain text.
- Prefer simple boxes and arrows over dense pseudo-art.
- Match the diagram type to the question: source, market, pathway, analysis, strategy, navigation, or update.
- If the document set needs multiple diagrams, plan them as a coherent series with shared labels, repeated visual logic, and a deliberate reading order.
- Do not diagram what a single sentence explains better.

## Output Expectations
- Create diagrams that survive terminal rendering.
- Keep labels short and meaningful.
- Support the prose instead of replacing it.
- When multiple diagrams appear in one document or folder, make them feel like one system rather than isolated sketches.
