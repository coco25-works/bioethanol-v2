# ASCII Layout Patterns

## Preferred building blocks
- boxes for concepts or layers
- arrows for flow
- stacked layouts for source and process logic
- side-by-side blocks for comparison

## Diagram series pattern
Use a connected series when one diagram cannot carry the full explanation clearly.

### Good reasons to use a series
- the reader needs a staged mental model across several files;
- one document moves from source to market to strategy;
- one folder needs a repeated visual language across multiple files;
- the same actors or concepts recur and should stay visually consistent.

### Series rules
- keep a stable visual vocabulary across the set;
- reuse the same names for the same concepts;
- keep the flow direction consistent where possible;
- make each diagram stand alone, but let the series build understanding cumulatively;
- avoid redrawing the same diagram with only cosmetic changes.

## Good use cases
- source stack
- market architecture
- pathway landscape
- analytical framework
- strategy decision logic
- repo navigation
- update propagation

## Rules
- keep line width reasonable;
- use consistent box width inside one diagram;
- avoid more than one crossing flow in small diagrams;
- follow the diagram with one or two sentences on what it shows.
- if using a series, note the sequence explicitly in prose, for example `This diagram shows the source layer; the next diagram shows the market layer`.
