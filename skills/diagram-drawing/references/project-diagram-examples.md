# Project Diagram Examples

## Source stack
Use for source-register or evidence-log files.

## Market architecture
Use for market, country, or jurisdiction reference files.

## Pathway landscape
Use for pathway, product-family, or workflow files.

## Analytical framework
Use for comparison, ranking, or option-evaluation files.

## Strategy decision logic
Use for strategy or recommendation files.

## Navigation and update flow
Use for QA, maintenance, or multi-step operations files.

## Connected diagram series example
Use a series when the reader needs to move through a layered system.

Example sequence:
1. source stack
2. market architecture
3. pathway landscape
4. analytical framework
5. strategy decision logic
6. update propagation

This pattern works well when a project has multiple document layers and the diagrams should connect conceptually from evidence through recommendation and maintenance.
