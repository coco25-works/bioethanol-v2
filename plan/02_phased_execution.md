# Phased Execution Plan

## Build Sequence

## Phase 0. Program Design
Create the working rules before any research drafting starts.

Outputs:
- scope and objectives
- research method
- evidence hierarchy
- fuel/feedstock taxonomy
- market-analysis framework
- strategy framework

Quality gate:
- the team can tell where every future claim should live.

## Phase 1. Source Harvest
Build the source register in separate jurisdiction files.

Outputs:
- one source-register file per jurisdiction
- source gap log

Quality gate:
- every priority jurisdiction has primary legal anchors logged.

## Phase 2. Country Reference Build
Convert the source logs into structured reference files.

Outputs:
- one country-reference file per jurisdiction

Quality gate:
- the same schema is used in every jurisdiction file;
- every material legal point traces back to a source-register entry.

## Phase 3. Biofuel Reference Build
Create pathway-centered files that synthesize treatment across markets.

Outputs:
- one file per major pathway;
- one file for certifications and chain of custody;
- one file for special feedstock groupings where needed.

Quality gate:
- each pathway file points back to the relevant country-reference files.

## Phase 4. Market Analysis Build
Produce comparative analysis based on the validated reference layer.

Outputs:
- demand-mechanism comparisons
- regional supply-potential analysis
- feedstock-to-market mapping
- market ranking outputs

Quality gate:
- every analysis claim is traceable to a country or pathway reference.

## Phase 5. Strategy Output Build
Write separate strategy files based on the analysis layer.

Outputs:
- executive strategy summary
- pathway strategy files
- market-entry strategy files
- producer-type strategy files
- partnership and risk files

Quality gate:
- every recommendation points back to supporting analysis and reference files.

## Phase 6. Final QA and Consolidation
Make the full set navigable and consistent.

Outputs:
- glossary
- acronyms
- legal citation index
- final source gap log
- QA checklist
- cross-reference cleanup and folder navigation hardening

Quality gate:
- the repository can be navigated by country, fuel, or strategy question without ambiguity.

## Phase 7. Update Operations and Maintenance
Turn the completed repository into a maintainable internal reference system with explicit refresh rules and change control.

Outputs:
- update policy
- regulatory and pathway watchlists
- change log
- versioning rules
- refresh cadence and trigger model

Quality gate:
- the repository can be refreshed without bypassing the evidence-first workflow.
