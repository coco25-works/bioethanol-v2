# Strategy Usefulness Checklist

This checklist defines whether the strategy layer is genuinely useful for a commercial / BD audience rather than merely well-written.

Use it in three places:
- during Phase 5 drafting;
- during Phase 6 final QA;
- during Phase 7 refresh cycles.

## Core Questions
Every major strategy file should let a BD user answer:
- Is this worth pursuing?
- Who would buy or monetize it?
- Why now?
- What small rule difference makes this stronger than a similar market?
- What must be true first?
- What would make us stop?
- What stronger alternative should we prefer if this one is weak?

## Checklist
### Demand and buyer logic
- The file states what creates demand.
- The file names the relevant buyer or counterparty classes.
- The file explains why those buyers would care now.
- The file explains what legal asymmetry makes those buyers or routes more attractive than the nearest comparable market.

### Commercial decision support
- The file includes explicit recommendation labels.
- The file includes commercial decision criteria.
- The file distinguishes between `Priority 1`, `Priority 2`, `Selective`, `Monitor`, and `No-go` in a way that is easy to apply.

### Gating and risk
- The file includes gating requirements.
- The file includes real stop conditions, not generic caveats.
- The file states what would change the conclusion.
- The file states whether the opportunity depends on a temporary transition window or a more durable rule asymmetry.

### Comparative usefulness
- The file makes clear whether a better market-pathway alternative exists.
- The file explains why this opportunity is stronger or weaker than competing options.
- The file identifies whether the advantage comes from timing, classification, proof recognition, administrative access, or value-capture design.

### Evidence and confidence
- The file cites the relevant upstream evidence base.
- The file makes evidence confidence visible where uncertainty matters.
- The file does not rely on hidden assumptions.
- The file makes the upstream asymmetry trail visible enough that a reviewer can verify the delta without inference.

## Pass Standard
A strategy file passes only if a BD user can use it to decide one of the following without needing to infer missing logic:
- pursue now;
- build capabilities first;
- monitor;
- reject.
